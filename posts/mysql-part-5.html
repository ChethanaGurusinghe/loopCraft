how to rename columns, and how to add.

You may remember that in Part 4, we learned the data insert part as the 6th query. That is only one way to insert data. Apart from the method taught in the previous part, there are two other ways to write the Insert query .
In the previous part (MySQL Part 4), we inserted only one row at a time. But we can insert data into multiple rows with one insert query..all we have to do is write the query by separating the rows with commas(,) in the same query we wrote earlier...

INSERT INTO <table name> VALUES (value1, value2, value3,...), (value1, value2, value3,...), (value1, value2, value3,...);

Eg :- INSERT INTO user VALUES ("U003","John",0123968745),("U004","Doe",0719875432),("U005","Jane",0916895742);

Okay, now you can insert data into multiple rows at once. But there is a small problem when you do this. Suppose there is a user named Nimal. He does not have a contact_no. Okay, the data in the column named contact_no for Nimal should be empty. Let's insert the data as we did before and see if we can do that. Since there is no contact_no, we write the query without providing that data.

INSERT INTO user VALUES ("U006","Nimal");

See what happens....

An error message has come up saying that the row we inserted does not match the number of columns in the table. Okay, so we inserted one less data...
In times like these (if we don't know the data for all the columns in the table), we need to specify which columns we are inserting data into. That is, we need to clearly state which columns in the table we are inserting data into.

INSERT INTO user (user_id, name) VALUES ("U006","Nimal");

This query is saying to insert U006, Nimal values ‚Äã‚Äãinto the user_id and name columns of the user table.. Since no data was inserted into contact_no, it is null ..

Okay, now you can create a db, create tables in it and store data. Imagine if you want to delete a table you created? There is a separate query for that...

(08.) DROP TABLE <table name>;

Eg :- DROP TABLE user;

This query will delete the table along with the data in the table you created. Now if you run the SHOW TABLES query, you will get an output like this. (SHOW TABLES; - You can see what tables are in the database)

In this case, it says empty set because I only created the user table in the database...deleting the user table means that there are no tables in the database. If there were other tables, they would be shown at this time.

You can delete the database just as you deleted the table...

(09.) DROP DATABASE <database name>;

Eg :- DROP DATABASE user_management_system;

Now we can check if the database we deleted was actually deleted by using SHOW DATABASES...

Once we have a table, we can make many changes to that table. Imagine that you have created a table called Employee. The columns are emp_id , name , emp_salary....

Okay, now that you've created this table, you need to add a new column called city. How do we do that?

(10.) ALTER TABLE <table name> ADD COLUMN <column name> <data type> <No of characters> ;

Eg :- ALTER TABLE employee ADD COLUMN city VARCHAR(30);

If you are adding a new column, the query should include information such as which table the new column is being added to, what the column name is, what the data type is, and how many characters it has...

What if I need to rename an existing column?

(11.) ALTER TABLE <table name> RENAME COLUMN <old column name> TO <new column name>;

Eg :- ALTER TABLE employee RENAME COLUMN city TO location;

When we write this query, we need to put both the name and the name that we want to rename from the current column name in the table...

If I want to rename the newly added column called city to location, I first need to tell you what is the name of the table to be renamed, what is the current column name of the column to be renamed (old name = city), and what name should it be renamed to? (new name = location)

Just like deleting a table or database, you can also delete columns in a table.

(12.) ALTER TABLE <table name> DROP COLUMN <column name>;

Eg :- ALTER TABLE employee DROP COLUMN location ;

If we want to make any changes to a column, row, or data within a table, we must first know the name of the table.
What is the name of the relevant table and what is the column name It is in the query.. (Renamed location after the newly added city)
